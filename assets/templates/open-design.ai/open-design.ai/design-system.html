Sobre o estúdio · Nº 02

Tratamos seu agente como um colaborador criativo, não uma caixa-preta.

Os agentes de programação mais fortes já vivem no seu laptop. Nós não enviamos um — nós os conectamos a um fluxo de design movido por skills que roda localmente com pnpm tools-dev, publica a camada web no Vercel e permanece BYOK em cada camada.

IV. Método / Loop•04 etapas, iterativo 005 / 008
Método · Nº 05

De sinais a sistemas.

+

Cada etapa é iterativa, visual e orientada a pesquisa — arquivos combináveis, não prompts opacos.

01

Detectar →

O daemon varre seu $PATH em busca de 12 agentes de programação e carrega automaticamente 132 skills + 150 sistemas na inicialização.

02

Descobrir →

O turno 1 é um formulário de perguntas — superfície, público, tom, escala, contexto de marca. Fechado em 30 segundos.

03

Direcionar →

Escolha uma de 5 direções visuais determinísticas. Paleta em OKLch, stack de fontes, dicas de postura de layout.

04

Entregar →

O agente escreve em disco, você visualiza num iframe isolado, e exporta HTML / PDF / PPTX / ZIP / Markdown.

Skills informam tudo. Arquivos tornam real.

VI·5. Perguntas frequentes•Respostas oficiais, sem enrolação de marketing 013 / 008
Perguntas frequentes do Open Design · Nº 06.5

Perguntas sobre Open Design, OpenDesign e a alternativa open-source ao Claude Design.

  1. 01O que é o Open Design?+

    Open Design é o workspace de design por IA open-source e oficial do projeto nexu-io/open-design. Ele transforma um agente de programação local — Claude Code, Codex, Cursor, Gemini CLI, OpenCode ou Qwen — em um motor de design movido por skills combináveis e sistemas DESIGN.md portáteis.

  2. 02O Open Design é oficial?+

    Sim. O projeto canônico vive em open-design.ai e o código-fonte está no GitHub em github.com/nexu-io/open-design. "Open Design", "OpenDesign", "open-design" e "Open Design AI" se referem todos ao mesmo projeto.

  3. 03Como o Open Design é diferente do Claude Design?+

    Claude Design é um produto hospedado preso a um único fornecedor. Open Design é local-first, open source sob Apache-2.0 e BYOK: você traz seu próprio agente, credenciais e sistema DESIGN.md.

  4. 04O Open Design roda localmente?+

    Sim. O app desktop, o daemon e o runtime de skills rodam na sua máquina. Os artefatos gerados ficam no diretório do seu projeto, em vez de serem forçados por uma nuvem de fornecedor.

  5. 05Quais agentes de programação o Open Design suporta?+

    O Open Design vem com 17 adaptadores BYOK de primeira mão prontos para uso: Claude Code, Codex, Cursor, Gemini CLI, GitHub Copilot CLI, Grok, Hermes, Kimi, Devin for Terminal, OpenCode, Qwen, DeepSeek, Pi, Mistral Vibe, Kiro, Kilo e Qoder. Qualquer adaptador que fale o mesmo protocolo de skills funciona — trocar de agente é uma mudança de configuração, não um redesign.

  6. 06Posso auto-hospedar ou fazer fork dele?+

    Sim. O código é Apache-2.0. Você pode fazer fork do repositório, editar skills, adicionar seus próprios sistemas DESIGN.md ou rodar o daemon nas suas próprias máquinas.

  7. 07Meus dados são enviados para Anthropic, OpenAI ou Google?+

    Apenas seu prompt e o contexto da skill vão para o provedor cujas chaves você traz (BYOK). O Open Design não tem servidor próprio — o daemon fala diretamente com o seu provedor. Os artefatos gerados ficam como arquivos no diretório do seu projeto, e não em qualquer nuvem de fornecedor.

  8. 08Posso usar o Open Design sem instalar a CLI ou o app desktop?+

    Hoje não. O Open Design é local-first por design — o mínimo é um daemon local mais um agente (Claude Code, Codex, Cursor, Gemini CLI ou um dos 17 adaptadores suportados). Um sandbox hospedado está no roadmap, mas não é prioridade: artefatos no seu repositório vencem documentos no banco de dados de outra pessoa.

  9. 09Quanto custa o Open Design?+

    O produto é gratuito e Apache-2.0 — não há assinatura do Open Design. Você paga os custos de API de qualquer provedor que usar (Anthropic, OpenAI, Google, Mistral, xAI, Moonshot, etc.), cobrados diretamente na sua conta. O BYOK mantém tanto as credenciais quanto o gasto do seu lado da linha.

  10. 10Posso auto-hospedar o Open Design no Vercel, Cloudflare ou no meu próprio servidor?+

    Sim. O daemon roda em qualquer lugar onde o Node 24 rode, e a landing page é um build estático em Astro que publica no Cloudflare Pages, Vercel ou Netlify sem alterações. Equipes com deploys compartilhados normalmente fixam o daemon numa máquina dentro da rede e apontam a CLI de cada desenvolvedor para ele.

  11. 11Como movo minha marca para o Open Design?+

    Solte um screenshot ou um export do Figma na UI web e peça ao seu agente para extrair uma marca para um arquivo DESIGN.md. Salve esse arquivo em design-systems/<sua-marca>/ no seu repositório; toda skill então renderiza nessa marca sem reprompt. /alternatives/claude-design/ descreve o mesmo fluxo em forma de passos.

  12. 12Posso trocar de agente sem refazer meu trabalho?+

    Sim. Skills e sistemas DESIGN.md são agnósticos a agente — o mesmo arquivo SKILL.md renderiza no Claude Code, Codex, Cursor, Gemini CLI, GitHub Copilot, Grok, Hermes, Qwen ou qualquer outro adaptador suportado. Trocar de agente é uma mudança de configuração no daemon, não um redesign.

  13. 13O que está no roadmap do Open Design?+

    O roadmap ao vivo está em docs/roadmap.md no repositório do GitHub, e as notas de release semanais saem via GitHub Releases. Temas principais para o próximo trimestre: mais adaptadores de agente, famílias de template mais ricas (3D, vídeo, áudio) e um modo opcional de daemon compartilhado para equipes de design.
